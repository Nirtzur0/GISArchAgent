# CI Manifest

## Current State
- Baseline CI workflow is committed at `.github/workflows/ci.yml`.
- Dedicated release workflow is committed at `.github/workflows/release.yml`.
- Trigger coverage:
  - `ci.yml`: `push` to `main`/`master` and all pull requests.
  - `release.yml`: tag pushes matching `v*` and manual `workflow_dispatch`.

## Baseline Workflow Coverage (`ci.yml`)
### Mandatory jobs
- `quality`: expanded quality gates with bounded debt burn
  - repo-wide ruff syntax/parse-focused lint gate (`E9`, `F63`, `F7`) excluding `project-prompts/`
  - formatting guard on actively maintained packet surfaces (`CMD-022`)
  - formatting guard on changed non-allowlisted Python files only (`CMD-030`) to prevent new style debt and burn down legacy debt when files are touched
  - debt baseline tracked in `scripts/quality_black_debt_allowlist.txt` and reviewed via `CMD-031`
  - dependency reproducibility/drift guard (`CMD-033`) validates lockfile coverage + docs sync
- `tests`: marker-based gates
  - `unit`
  - `integration`
  - `e2e`
  - `data_contracts`
- `prompt-pack-integrity`
  - prompt manifest check
  - prompt system integrity check

## Release Workflow Coverage (`release.yml`)
### Mandatory jobs
- `release-validation`
  - verifies release artifacts are present
  - validates tag/changelog semantic coherence
  - runs marker gates (`unit`, `integration`, `e2e`, `data_contracts`)
  - runs prompt-pack integrity checks
- `publish-github-release`
  - runs on tag refs only and creates a GitHub Release with generated notes

## PR and Release Guardrails
Any PR changing source/tests/CI should also update:
- `docs/implementation/00_status.md`
- relevant checklist/report files in `docs/implementation/`

Any release tag (`v*`) should only be pushed after:
- release readiness checklist is updated (`docs/implementation/checklists/06_release_readiness.md`)
- `CHANGELOG.md` includes release notes
- `CMD-041` guardrail passes for onboarding/reference artifact-link citations
- command-map verification is green locally or in CI

## Known Gaps / Next CI Steps
- Full-repo lint syntax coverage is now enforced; full-repo formatting remains a phased debt burn.
- Debt-burn sequencing for formatting:
  - Phase 1 (completed): enforce formatting on changed non-allowlisted Python files (`CMD-030`) plus maintained critical surfaces (`CMD-022`).
  - Phase 2 (current): migrate one legacy module group at a time into always-enforced `CMD-022`.
    - latest migration completed: bounded `src/infrastructure/*` group
      (`__init__.py`, `factory.py`, `repositories/chroma_repository.py`, `services/cache_service.py`, `services/document_service.py`, `services/llm_service.py`, `services/vision_service.py`) promoted to always-enforced surfaces.
    - debt baseline (`CMD-031`) reduced from `73` lines to `66` lines in `scripts/quality_black_debt_allowlist.txt`.
  - Phase 3 (target): replace phased guardrails with full-repo `black --check`.
- Dependency reproducibility baseline:
  - `requirements.lock` is the deterministic environment lock artifact.
  - `docs/reference/dependencies.md` is generated from manifest + lock state.
  - `CMD-033` enforces that lockfile covers direct requirements and docs inventory is up to date.
- Optionally add stronger release semantics checks (for example: section content/ordering validation).
- Keep optional live-network tests gated (`RUN_NETWORK_TESTS=1`) and blocked in CI unless explicitly overridden (`RUN_NETWORK_ALLOW_CI=1`) in a dedicated rehearsal workflow.
